Support for Easy JSON Prettier

Last updated: June 19, 2026

Easy JSON Prettier is a Safari Web Extension that detects raw JSON responses by looking at a page's actual content, and renders them as a readable, collapsible, syntax-highlighted view. This page covers how to turn it on, how to use it, and how to get help if something isn't working.

Getting started

  1. Open Safari → Settings → Extensions and turn on JSON Formatter.
  2. Visit any page that returns a raw JSON response (for example, opening an API endpoint directly in a tab). The page will automatically switch to the formatted, collapsible view.
  3. Click the Easy JSON Prettier icon in Safari's toolbar to see whether JSON was detected on the current page and to toggle formatting on or off.

Using the formatted view

  • Expand All / Collapse All — open or close every node in the tree at once.
  • Filter keys or values — type in the search box to highlight matching keys and values and automatically expand the tree down to them.
  • Raw — switch to the original, unformatted text of the response.
  • Copy — copies the full, pretty-printed JSON to your clipboard.
  • Save — downloads the JSON to a file.

Very large documents start partially collapsed so the page stays responsive; use Expand All if you want to see everything at once.

Turning formatting on or off

Use the toolbar popup to control Easy JSON Prettier:

  • Enabled everywhere turns the extension on or off globally.
  • Enabled on <site> turns formatting on or off just for the current site, leaving every other site unaffected.

These settings are stored locally in Safari and are never synced or transmitted anywhere.

Frequently asked questions

A page that looks like JSON isn't being formatted. Why?

Easy JSON Prettier only reformats a page when its entire body is a single valid JSON document — it never looks at the URL, file extension, or HTTP headers. If the page also contains HTML markup, multiple top-level elements, or any non-JSON text, it's left completely untouched. Also check the toolbar popup: it tells you whether JSON was detected and whether formatting is currently on for that site.

A normal HTML page got reformatted unexpectedly.

This shouldn't happen — detection only triggers when a page's rendered body is exactly one JSON value and nothing else. If you find a page where this isn't the case, please get in touch (see below) with the URL so it can be fixed.

I turned off formatting for a site, but it's still showing the raw view I expect to be formatted.

Toggling "Enabled on <site>" off shows the original, unformatted response for that host. Turn it back on in the popup, or use "Enabled everywhere" to restore formatting globally.

Does Easy JSON Prettier send my data anywhere?

No. It runs entirely on your device, makes no network requests, and has no analytics, crash reporting, or third-party SDKs. See the Privacy Policy for details.

Why does the extension need access to all websites?

Because JSON can be returned from any domain, Easy JSON Prettier needs to be able to check the content of any page to decide whether it's a raw JSON response. It only reads the page's own rendered text to make that decision — see the Privacy Policy for the full explanation.

The Copy or Save button didn't do anything.

Copy requires clipboard permission in Safari; if a permission prompt appeared, allow it and try again. Save opens your browser's normal file-save dialog. If either still doesn't work, please report it — see below.
